# Domain Pack Research Evidence

Last updated: 2026-06-05

This note summarizes the current evidence for the mainline research question:

```text
Can a closed predicate domain, built from a small seed set, transfer hard-clean
to unseen messy official documents in the same family under strict governance?
```

It is not a product claim and not a general-document accuracy claim. It is a
checkpoint for the technical thesis after the sign-clean reset.

## Claim-Bearing Gates

A domain-pack result is claim-bearing only when the relevant run satisfies the
gate named for that result:

- closed profile registry with lens-scoped offered signatures;
- N>=3 same-condition compiles with support>=2 for supported rows;
- 0 supported forbidden rows;
- registered signatures only;
- atom-shape pass;
- lens-scope pass;
- carrier value-domain pass where applicable;
- sign-clean audit pass for current code;
- no source-record prose matching, query-text routing, or fixture vocabulary in
  the winning path.

Targeted replay is mechanism evidence, not a corpus claim. A composed result is
diagnostic unless a fresh same-condition bundle reproduces it.

## Current Research Verdict

The strongest current evidence for the headline question is not FDA's largest
worksheet, but the cross-family pattern:

- The governance result is stronger than the recall result. Dense SEC compile
  substitutions stayed inside the closed language with `0` supported forbidden
  rows and clean atom/lens/signature gates, but both Gemma 4 12B Q4 and Qwen
  27B Q4 landed at `10/12` on transfer_003 where the local Qwen MoE lane
  wobbles in an `11-13/13` band after axis repair. The current claim is
  therefore: closed domains constrain the language across model paths, while
  recall and role semantics remain model/path sensitive. Cross-model compile
  robustness is not established.
- SEC shows that a small closed skeleton pack can transfer across three unlike
  same-family official documents under hard governance, but the repaired
  axis-clean schema has not yet reproduced the old historical number.
  The standing manifest now lands at seed 13/13, transfer_001 5/13,
  transfer_002 11/12, and transfer_003 11/13 after the transfer_001
  current-mainline full-bundle negative retest replaced the older favorable
  11/13 breadth root and a transfer_002 current-mainline retest preserved the
  older 11/12 score under the current root. A later omission-accountability guard cleaned invalid
  omission rows without lifting support; a later 2026-06-05 transfer_003
  refresh landed at a favorable 13/13 with 0 supported forbidden, but that is
  one point inside the observed repaired-schema band. SEC is a boundary-aware
  case study, not a pristine anchor.
- FDA shows the richer case-study path and the failure boundary: one unlike
  transfer cell is clean under current gates, and two fresh current-pack
  transfer cells are governance-clean but incomplete. Transfer_003 no longer
  needs the older archived integrity-fail story to make the point; its current
  clean run lands at 19/26 with every unsupported expected row in
  `fda_violation_detail/5`.
- NTSB shows the same boundary from another family: wrapper, chronology,
  vehicles, and conditions are easier to stabilize than findings, causal prose,
  and dense relational substance.
- OSHA adds fourth-family corroboration from retained public fixtures: the
  standing seed and first unlike transfer cells now fully support their expected
  skeleton/table facts under clean governance, while the previous source-true
  FTA total-penalty extra has been folded into the oracle and independently
  accepted by returned source-only review. The larger long-table/mixed-source
  diagnostics remain boundary evidence.
- State-AG settlement/AOD work has started as a fifth domain seed, not as a
  transfer claim. The current fresh all-lens seed rerun is 17/27 with 0
  supported forbidden rows, 57 unexpected same-signature facts, clean
  atom/lens/signature governance, and a clean carrier value-domain audit. None
  of the unexpected rows survives support>=2. Its
  value is process evidence: a narrow mapper-role gate correction, finite
  typed-alias normalization, and value-domain enforcement helped make the
  boundary visible, but citation subsection specificity, event-subject naming,
  counsel/registered-agent attachment, monetary payment support, and
  obligation-oracle design remain unresolved before transfer testing.

This is a publishable research shape only if the claim stays narrow:

```text
Closed, lens-scoped predicate domains can stabilize recurring official-document
anatomy under strict governance; open-ended substance remains an abstention or
lower-tier boundary unless a compact domain layer reproduces on unlike
documents.
```

```text
A small, closed, lens-scoped predicate pack can stabilize recurring skeleton
anatomy for messy official documents within the same family, under hard
governance, across retained unlike transfer fixtures.
```

The strongest current positive pattern is not a single SEC score but the
cross-family recurrence: skeleton, categorical, chronology, and table anatomy
are easier to stabilize than open findings, causal prose, dense relational
substance, wrapper semantics, treatment attachment, and blank-value policy. SEC
now contributes both a positive skeleton signal and a boundary signal around
item-treatment attachment under an axis-clean schema.

## Judge / Oracle Boundary

The current domain-pack measurements in this note are typed-fact transfer
measurements: expected facts supported at N>=3/support>=2, forbidden facts not
supported, and integrity gates clean. They are not QA reference-judge exact
rates, so the answer judge cannot inflate these numbers.

If a future LLM-judged QA metric becomes claim-bearing, it must pass the
reference-judge null-control audit first. Deterministic compile-fact QA, like
the FDA v2 bundle, instead has to pass typed-plan replay and redaction replay:

```text
script: scripts/audit_reference_judge_null_controls.py
controls:
  empty_evidence: true reference answer with no evidence
  wrong_reference: redacted typed evidence with a different same-fixture answer
  wrong_reference question text is replaced with a neutral null-control
    instruction, so the judge cannot follow the original question while
    ignoring the swapped reference
blocking condition:
  any exact verdict in either null control
```

```text

  root: C:\prethinker_tmp_archive\reference_judge_null_controls_20260604\sample3_pre_harness_fix_blocked
fix:
  wrong_reference controls now neutralize the original question/utterance and
  blank model/projected decision text before calling the reference judge
post-fix sample3:
  6 product-exact rows sampled, 12 controls, 0 exact null verdicts
  root: C:\prethinker_tmp_archive\reference_judge_null_controls_20260604\sample3_post_harness_fix_pass
```

This is a harness-governance result, not a new FDA score. It shows the
null-control can catch a judge-facing question leak, and that the audit itself
must be caged against the same scoreboard pressure it is meant to detect.

Oracle changes also remain governed by source-only review: a correction is not
claim-bearing unless a reviewer who did not see the model output can defend it
from the source package alone.

## What This Does Not Support

Do not claim:

- 90%+ general Prethinker QA accuracy;
- arbitrary-domain document understanding;
- event-substance extraction for SEC filings;
- product readiness;
- self-serve domain-pack induction;
- that a single transfer fixture proves a domain pack transfers;
- that composed historical runs are equivalent to fresh same-condition bundles;
- any LLM-judged QA exact-rate claim that has not passed judge null controls.
