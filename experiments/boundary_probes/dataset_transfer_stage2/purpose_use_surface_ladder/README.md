# Purpose/Use Surface Ladder

This probe tests whether a source-stated use or purpose remains queryable when
the same sentence neighborhood also contains lower-level properties,
measurements, instruments, or methods.

The intended architecture question is fixture-free:

- Can a QA pass recover the task/purpose phrase for "used for" questions?
- Does the selector over-prefer adjacent measurement/property rows when the
  question asks for the broader source-stated use?
- Are explicit negative-purpose statements preserved without inventing a
  positive use?
